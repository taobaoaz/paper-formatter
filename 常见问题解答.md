## 1. 安装与启动

### Q1.1: 双击启动程序没反应，怎么办？

**问题描述：** 双击 `启动程序.bat` 后没有任何反应。

**可能原因：**
1. Python 未安装
2. Python 未添加到系统 PATH
3. 依赖包缺失
4. 权限问题

**解决方法：**

**步骤 1：检查 Python 是否安装**
```bash
# 打开命令提示符（Win + R，输入 cmd）
python --version
```

如果显示版本号（如 `Python 3.10.0`），说明已安装。  
如果显示"'python' 不是内部或外部命令"，说明未安装或未配置 PATH。

**步骤 2：安装 Python**
```
1. 前往 https://www.python.org/downloads/
2. 下载 Python 3.8 或更高版本
3. 运行安装程序
4. ⚠️ 重要：勾选 "Add Python to PATH"
5. 点击 "Install Now"
```

**步骤 3：手动安装依赖**
```bash
cd paper-formatter
pip install -r requirements.txt
```

**步骤 4：手动启动**
```bash
python main.py
```

---

### Q1.2: 提示"找不到模块"或"ImportError"，怎么办？

**问题描述：** 启动时提示类似错误：
```
ModuleNotFoundError: No module named 'PyQt5'
```

**解决方法：**

**方法 1：安装所有依赖**
```bash
pip install -r requirements.txt
```

**方法 2：使用国内镜像（更快）**
```bash
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

**方法 3：逐个安装**
```bash
pip install PyQt5 python-docx openpyxl chardet requests Pillow pyyaml
```

**方法 4：升级 pip 后重试**
```bash
python -m pip install --upgrade pip
pip install -r requirements.txt
```

---

### Q1.3: 如何在没有网络的电脑上使用？

**问题描述：** 需要在内网或无网络环境的电脑上使用软件。

**解决方法：**

**步骤 1：在有网络的电脑上准备**
```bash
# 下载所有依赖包
pip download -r requirements.txt -d packages
```

**步骤 2：复制文件**
```
复制以下内容到 U 盘：
- paper-formatter 整个文件夹
- packages 文件夹（下载的依赖包）
```

**步骤 3：在目标电脑上安装**
```bash
# 在目标电脑上
cd paper-formatter
pip install -r requirements.txt --no-index --find-links=packages
python main.py
```

---

### Q1.4: 软件支持 Mac 或 Linux 吗？

**问题描述：** 想在 Mac 或 Linux 系统上使用。

**回答：**

**当前版本：** 主要支持 Windows

**在 Mac/Linux 上使用：**
```bash
# 1. 安装依赖（相同）
pip install -r requirements.txt

# 2. 修改启动脚本（创建 start.sh）
#!/bin/bash
python main.py

# 3. 运行
chmod +x start.sh
./start.sh
```

## 2. 文档处理

### Q2.1: 支持哪些文件格式？

**支持的输入格式：**
- ✅ `.docx` - Microsoft Word 2007+
- ✅ `.tex` - LaTeX 文档

**不支持的格式：**
- ❌ `.doc` - 旧版 Word 格式
- ❌ `.wps` - WPS 文档
- ❌ `.pages` - Apple Pages

**解决方法（对于不支持的格式）：**
```
1. .doc 文件：用 Word 打开，另存为 .docx
2. .wps 文件：用 WPS 打开，另存为 .docx
3. .pages 文件：用 Pages 导出为 .docx
```

---

### Q2.2: 导入文档后显示空白或乱码，怎么办？

**问题描述：** 导入 Word 文档后，内容显示为空白或乱码。

**可能原因：**
1. 文件编码问题
2. 文件损坏
3. 特殊格式不兼容

**解决方法：**

**方法 1：检查文件完整性**
```
1. 用 Microsoft Word 打开文件
2. 确认内容正常显示
3. 另存为新文件
4. 重新导入
```

**方法 2：简化文档格式**
```
1. 复制所有内容
2. 粘贴到新建文档
3. 选择"只保留文本"
4. 重新应用基本格式
5. 保存后导入
```

**方法 3：检查特殊字符**
```
1. 查找特殊符号或表情
2. 删除或替换为标准字符
3. 保存后重试
```

---

### Q2.3: 大文件（>50MB）处理很慢，怎么办？

**问题描述：** 处理大文件时速度很慢或卡死。

**解决方法：**

**方法 1：分割文档**
```
1. 按章节分割成多个小文件
2. 分别处理每个文件
3. 最后合并
```

**方法 2：关闭其他程序**
```
1. 关闭浏览器
2. 关闭其他 Office 文档
3. 释放内存
```

**方法 3：增加虚拟内存**
```
1. 右键"此电脑" → 属性
2. 高级系统设置 → 高级 → 性能设置
3. 高级 → 虚拟内存 → 更改
4. 设置为物理内存的 1.5-2 倍
```

**方法 4：升级硬件**
```
- 增加内存到 8GB 或更高
- 使用 SSD 硬盘
```

---

### Q2.4: 如何批量处理多个文档？

**问题描述：** 需要一次性处理几十篇论文。

**解决方法：**

**方法 1：使用批量处理功能**
```
1. 点击 "工具" → "批量处理"
2. 选择包含所有文档的文件夹
3. 选择输出文件夹
4. 选择处理方式
5. 点击 "开始处理"
```

**方法 2：使用命令行（高级）**
```bash
# 创建批处理脚本 process_all.bat
@echo off
for %%f in (*.docx) do (
    echo Processing %%f
    python main.py --batch %%f
)
pause

# 运行脚本
process_all.bat
```

## 3. 模板使用

### 3.0 封面和声明页配置

### Q3.0.1: 如何配置封面和声明页？

**问题描述：** 想添加或修改论文的封面和声明页。

**解决方法：**

**步骤 1：打开配置器**
```
工具 → 封面和声明页配置
```

**步骤 2：配置封面**
```
1. 勾选 "启用封面"
2. 填写学校名称
3. 选择论文类型
4. 填写封面字段信息
```

**步骤 3：配置声明页**
```
1. 勾选 "启用声明页"
2. 选择声明类型
3. 勾选 "显示签名区域"
4. （可选）填写自定义声明内容
```

**步骤 4：保存配置**
```
1. 点击 "💾 保存配置"
2. 选择保存位置
3. 保存为 .json 文件
```

## 4. 格式化问题

### Q4.1: 中文字体显示为方框或乱码，怎么办？

**问题描述：** 文档中的中文显示为方框 □□□ 或乱码。

**可能原因：**
1. 系统缺少中文字体
2. 字体名称不匹配
3. 字体文件损坏

**解决方法：**

**方法 1：安装中文字体**
```
1. 下载中文字体包
   - 宋体 (SimSun)
   - 黑体 (SimHei)
   - 楷体 (KaiTi)
2. 右键字体文件 → 安装
3. 重启软件
```

**方法 2：替换为可用字体**
```
1. 设置 → 字体设置
2. 选择已安装的中文字体
3. 应用并保存
```

**方法 3：检查 Word 字体替换**
```
1. 用 Word 打开文档
2. 文件 → 选项 → 保存
3. 取消 "在文件中嵌入字体"
4. 保存后重试
```

---

### Q4.2: 行距设置后没有变化，怎么办？

**问题描述：** 设置了 1.5 倍行距，但文档看起来没有变化。

**可能原因：**
1. 段落设置了固定行距
2. 文档使用了样式定义
3. Word 缓存问题

**解决方法：**

**方法 1：清除固定行距**
```
1. 全选文档 (Ctrl + A)
2. 右键 → 段落
3. 行距选择 "多倍行距"
4. 设置值设为 1.5
5. 确定
```

**方法 2：更新样式**
```
1. 右键 "正文" 样式
2. 选择 "更新以匹配所选内容"
3. 应用到全文
```

**方法 3：重新应用模板**
```
1. 清除所有格式
2. 重新应用模板
3. 保存为新文件
```

---

### Q4.3: 页眉页脚设置后不显示，怎么办？

**问题描述：** 设置了页眉页脚，但打印预览看不到。

**可能原因：**
1. 页眉页脚高度设置不当
2. 不同节设置不同
3. 首页不同设置

**解决方法：**

**步骤 1：检查页眉页脚高度**
```
1. 设置 → 页面设置
2. 页眉高度：建议 1.5cm
3. 页脚高度：建议 1.75cm
```

**步骤 2：检查分节符**
```
1. 查看 → 显示编辑标记
2. 检查是否有分节符
3. 确保每节都设置了页眉页脚
```

**步骤 3：取消"首页不同"**
```
1. 双击页眉进入编辑
2. 取消勾选 "首页不同"
3. 应用到所有节
```

## 5. 参考文献

### Q5.1: 参考文献格式化后顺序错了，怎么办？

**问题描述：** 格式化后参考文献顺序混乱。

**解决方法：**

**方法 1：按出现顺序排序**
```
1. 工具 → 参考文献管理
2. 选择 "按出现顺序排序"

```

**方法 2：手动调整**
```
1. 使用参考文献编辑器
2. 拖拽调整顺序
3. 保存更改
```

**方法 3：检查引用标记**
```
1. 查看正文中的引用标记
2. 确保顺序正确
3. 更新参考文献列表
```

---

### Q5.2: 英文参考文献格式不对，怎么办？

**问题描述：** 英文文献格式化后不符合要求。

**可能原因：**
1. 格式标准选择错误
2. 作者姓名格式问题
3. 期刊名缩写问题

**解决方法：**

**方法 1：选择正确的格式标准**
```
- 工科用 IEEE
- 理科用 APA
- 文科用 MLA
- 中文用 GB/T 7714
```

**方法 2：检查作者姓名**
```
正确格式（IEEE）：
J. Smith, "Title," Journal, 2023.

错误格式：
Smith John, "Title," Journal, 2023.
```

**方法 3：使用文献管理工具**
```
1. 用 EndNote/Zotero 管理文献
2. 导出为正确格式
3. 导入到软件
```

---

### Q5.3: 如何从知网/Google Scholar 导入参考文献？

**问题描述：** 想快速导入文献。

**解决方法：**

**从知网导入：**
```
1. 在知网找到文献
2. 点击 "引用" 或 "导出"
3. 选择 "GB/T 7714" 格式
4. 复制引用文本
5. 在软件中粘贴导入
```

**从 Google Scholar 导入：**
```
1. 在 Google Scholar 找到文献
2. 点击引号图标 "
3. 选择 "BibTeX" 或 "EndNote"
4. 复制引用文本
5. 在软件中导入
```

```
1. 将大文档分成小文档
2. 分别处理
```
