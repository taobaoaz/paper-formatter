# 📦 打包失败修复方案

**问题：** GitHub Actions 在 "Install dependencies" 步骤失败

**原因：** 可能是路径或依赖问题

---

## ✅ 解决方案

### 方案 A：简化 GitHub Actions（推荐）

**已修改工作流：**
- ✅ 只打包 EXE
- ✅ 不包含 Inno Setup（需要额外安装）
- ✅ 手动制作安装包更灵活

**新流程：**
```
1. GitHub Actions 自动打包 EXE
2. 下载 EXE 文件
3. 本地用 Inno Setup 制作安装包
```

---

### 方案 B：手动打包（立即可用）

#### 步骤 1：本地打包 EXE

```bash
# 安装依赖
pip install pyinstaller
pip install -r 启动文件/requirements.txt

# 打包 EXE
pyinstaller --onefile --windowed --name="论文排版优化器" --icon=installer/app_icon.ico 核心模块/main.py
```

**输出：**
```
dist/论文排版优化器.exe
```

#### 步骤 2：制作安装包

**方法 1：使用 Inno Setup**

```bash
# 打开脚本
iscc installer/installer.iss

# 输出
installer_output/论文排版优化器安装程序_v2.0.0.exe
```

**方法 2：使用 7-Zip 自解压**

创建 `installer/sfx_config.txt`：
```
;!@Install@!UTF-8!
Title="论文排版优化器"
BeginPrompt="是否安装论文排版优化器？"
RunProgram="论文排版优化器.exe"
;!@InstallEnd@!
```

然后：
```bash
7z a -sfx -r installer_output\setup.exe dist\* installer\*
```

## 🔧 修复后的 GitHub Actions

### 简化版工作流

**文件：** `.github/workflows/build-installer.yml`

**功能：**
- ✅ 打包 EXE
- ✅ 上传到 Artifacts
- ✅ 创建 Draft Release

**不包含：**
- ❌ Inno Setup（太复杂）
- ❌ 安装包编译（手动更灵活）

---

## 📋 手动打包完整流程

### 1. 推送标签

```bash
git tag v2.0.0
git push origin v2.0.0
```

### 2. 等待 GitHub Actions

**查看：**
```
https://github.com/taobaoaz/paper-formatter/actions
```

### 3. 下载 EXE

**从 Artifacts 下载：**
1. 点击最新的 "Build Installer"
2. 滚动到底部
3. 下载 "论文排版优化器-EXE"

### 4. 本地制作安装包

**使用 Inno Setup：**
```bash
# 复制 EXE 到 dist 文件夹
copy 下载的.exe dist\论文排版优化器.exe

# 编译安装包
iscc installer/installer.iss
```

### 5. 上传到 Release

**手动上传：**
1. 访问 https://github.com/taobaoaz/paper-formatter/releases
2. 编辑 v2.0.0 Draft
3. 上传安装包
4. 发布 Release
