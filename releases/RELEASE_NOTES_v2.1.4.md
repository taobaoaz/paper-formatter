## 🎯 功能特点

### 撤销管理器

**UndoManager 类：**
- 最大撤销步数：50 步
- 自动保存历史到文件
- 支持监听器模式（状态变化通知）
- 支持序列化/反序列化

**UndoAction 数据类：**
- action_type: 操作类型
- file_path: 操作的文件路径
- description: 用户友好的描述
- timestamp: 操作时间戳
- before_state: 操作前状态
- after_state: 操作后状态
- undo_data: 撤销所需的完整数据

### 持久化

**历史保存：**
- 自动保存到 `.config/undo_history.json`
- 程序重启后自动加载历史
- 支持手动清空历史

### 配置管理

**验证配置：**

## 📝 使用场景

### 撤销/重做场景

**场景 1：格式化后不满意**
```
1. 对文档进行格式化
2. 发现效果不理想
3. 按 Ctrl+Z 撤销格式化
4. 重新选择模板或调整参数
```

**场景 2：多次尝试不同模板**
```
1. 应用模板 A → 不满意 → Ctrl+Z 撤销
2. 应用模板 B → 不满意 → Ctrl+Z 撤销
3. 应用模板 C → 满意 → 保存
```

**场景 3：误操作恢复**
```
1. 不小心应用了错误的配置
2. 按 Ctrl+Z 撤销
3. 或者按 Ctrl+Y 重做（如果撤销过头了）
```

---

### 配置管理场景

**场景 4：配置验证**
```
1. 修改了大量配置参数
2. 运行配置验证
3. 查看验证报告，修复问题
4. 保存配置
```

**场景 5：配置备份与恢复**
```
1. 在修改重要配置前备份
2. 修改配置
3. 如果不满意，从备份恢复
```

**场景 6：配置对比**
```
1. 导出当前配置
2. 导入学校提供的标准配置
3. 比较差异
4. 根据差异调整配置
```

**场景 7：自动修复**
```
1. 导入外部配置（可能有错误）
2. 运行自动修复
3. 查看修复报告
4. 保存修复后的配置
```

## 🐛 已知限制

### 当前版本限制

1. **撤销范围：** 目前主要记录操作历史，实际撤销格式化需要与 formatter 模块深度集成（计划 v2.1.5）
2. **内存占用：** 大量操作历史可能占用较多内存（已限制最多 50 步）
3. **跨会话撤销：** 虽然历史会持久化，但跨会话的撤销可能需要重新加载文件

### 未来改进方向

1. **深度集成：** 与 formatter 模块集成，实现真正的「撤销格式化」
2. **增量保存：** 只保存变化的部分，减少内存占用
3. **快照功能：** 支持手动创建快照，快速恢复到特定状态
