# 📘 封面和声明页配置指南

**让你的论文包含完整的封面和声明页！**

---

## 📋 功能概述

封面和声明页配置器用于管理论文的前置页面，包括：

- 📘 **封面** - 学校名称、论文题目、作者信息等
- 📋 **独创性声明** - 学术诚信声明
- 📜 **版权使用授权书** - 版权授权
- 🔒 **保密论文声明** - 保密论文专用

---

## 🚀 快速开始

### 步骤 0：上传学校模板（推荐）

**如果你的学校提供了 Word 模板：**

```
1. 点击 "📂 上传模板并选择页面"
2. 选择学校的 Word 模板文件 (.docx)
3. 在弹出的页面选择器中：
   - 选择封面页（通常是第 1 页）
   - 选择声明页（通常是第 2 页）
4. 点击 "✅ 确定"
```

**效果：**
```
✓ 已上传：XX 大学论文模板.docx
封面：第 1 页 | 声明：第 2 页
```

**优势：**
- ✅ 完全符合学校格式要求
- ✅ 自动提取校徽、表格等元素
- ✅ 保留原模板的所有格式

### 步骤 1：打开配置器

在主界面点击：
```
工具 → 封面和声明页配置
```

或使用快捷键：
```
Ctrl + Shift + C
```

### 步骤 2：配置封面

#### 基本设置

```
┌─────────────────────────────────────┐
│  📘 封面配置                        │
├─────────────────────────────────────┤
│  [✓] 启用封面                       │
│                                     │
│  学校名称：[XX 大学        ]        │
│  论文类型：[本科毕业论文 ▼]        │
│  [✓] 显示校徽（如有）              │
└─────────────────────────────────────┘
```

**配置项说明：**

| 选项 | 说明 | 示例 |
|------|------|------|
| **启用封面** | 是否添加封面 | ✓ |
| **学校名称** | 学校全称 | "清华大学" |
| **论文类型** | 学位类型 | 本科/硕士/博士/课程 |
| **显示校徽** | 是否预留校徽位置 | ✓ |

#### 填写封面字段

```
┌─────────────────────────────────────┐
│  📝 封面字段信息                    │
├─────────────────────────────────────┤
│  论文题目：[基于深度学习的图像识别  ]│
│  作者姓名：[张三                  ]│
│  学号：    [202012345             ]│
│  专业：    [计算机科学与技术      ]│
│  指导教师：[李四 教授             ]│
│  完成日期：[2024 年 6 月            ]│
└─────────────────────────────────────┘
```

**必填字段：**
- ✅ 论文题目
- ✅ 作者姓名
- ✅ 学号

**选填字段：**
- ⚪ 专业
- ⚪ 指导教师
- ⚪ 完成日期

**提示：** 留空的字段在生成时不会显示。

### 步骤 3：配置声明页

#### 选择声明类型

```
┌─────────────────────────────────────┐
│  📋 声明页配置                      │
├─────────────────────────────────────┤
│  [✓] 启用声明页                     │
│                                     │
│  声明类型：[独创性声明 ▼]          │
│  [✓] 显示签名区域                   │
└─────────────────────────────────────┘
```

**声明类型说明：**

| 类型 | 适用场景 | 内容 |
|------|----------|------|
| **独创性声明** | 所有学位论文 | 学术诚信声明 |
| **版权使用授权书** | 需要授权学校使用 | 版权声明 |
| **独创性声明 + 授权书** | 完整版本 | 两者合并 |
| **保密论文声明** | 涉密论文 | 保密声明 |

#### 自定义声明内容（可选）

```
┌─────────────────────────────────────┐
│  📄 自定义声明内容（可选）          │
├─────────────────────────────────────┤
│  本人郑重声明：...                  │
│  （在此输入自定义声明内容）         │
│                                     │
│  [如果不填写，将使用标准声明模板]   │
└─────────────────────────────────────┘
```

**使用自定义内容的场景：**
- 学校有特殊声明要求
- 需要添加额外说明
- 保密论文特殊要求

### 步骤 4：保存配置

#### 保存为新配置

```
1. 点击 "💾 保存配置"
2. 选择保存位置
3. 输入文件名（如 "清华大学本科封面配置.json"）
4. 点击 "保存"
```

#### 加载已有配置

```
1. 点击 "📂 加载配置"
2. 选择之前保存的 .json 文件
3. 配置自动填充
```

保存的配置文件是 JSON 格式：

```json
{
  "cover": {
    "enabled": true,
    "type": "simple",
    "school_name": "清华大学",
    "thesis_type": "本科毕业论文",
    "show_logo": true,
    "fields": {
      "论文题目": "基于深度学习的图像识别研究",
      "作者姓名": "张三",
      "学号": "202012345",
      "专业": "计算机科学与技术",
      "指导教师": "李四 教授",
      "完成日期": "2024 年 6 月"
    }
  },
  "declaration": {
    "enabled": true,
    "type": "独创性声明",
    "show_signature": true,
    "custom_content": ""
  }
}
```

## 💡 使用技巧

### 技巧 1：为不同学校保存配置

**场景：** 帮助多个学校的学生格式化论文

**方法：**
```
保存多个配置文件：
- 清华大学_本科封面配置.json
- 北京大学_硕士封面配置.json
- 浙江大学_博士封面配置.json

使用时加载对应配置即可。
```

### 技巧 2：批量处理时复用配置

**场景：** 处理整个班级的论文

**方法：**
```
1. 配置好封面和声明页
2. 保存配置
3. 批量处理时加载同一配置
4. 只需修改作者信息
```

### 技巧 3：签名处理

**声明页需要手写签名：**

```
1. 软件生成时留空签名区域
2. 打印论文
3. 手写签名
4. 扫描提交

注意：签名区域已预留，不要填写。
```

### 技巧 4：使用自定义声明

**学校有特殊要求时：**

```
1. 获取学校声明原文
2. 粘贴到 "自定义声明内容"
3. 保存配置
4. 生成时使用自定义内容
```

## ❓ 常见问题

### Q1: 封面生成后格式不对，怎么办？

**可能原因：**
- 学校有特殊格式要求
- 字段顺序不对

**解决方法：**
```
方法 1：使用自定义模板
  - 上传学校的 Word 封面模板
  - 软件自动填写

方法 2：手动调整
  - 生成后在 Word 中微调
  - 保存为新模板
```

### Q2: 声明页的日期能自动填写吗？

**回答：** 可以。

**设置：**
```
在 "完成日期" 字段填写：
- 具体日期："2024 年 6 月"
- 或留空，打印时手写
```

### Q3: 可以只加封面，不加声明页吗？

**回答：** 可以。

**操作：**
```
取消勾选 "启用声明页" 即可。
```

### Q4: 配置文件可以分享吗？

**回答：** 可以！

**方法：**
```
1. 保存配置文件（.json）
2. 发送给同学
3. 同学加载配置即可使用
```

### Q5: 配置保存在哪里？

**默认位置：**
```
templates/covers/
├── tsinghua_cover.json
├── peking_cover.json
└── zhejiang_cover.json
```

**建议：** 保存在容易找到的位置，如：
```
D:/论文模板/封面配置/
```
