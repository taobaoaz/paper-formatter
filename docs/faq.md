# 常见问题（FAQ）

本文档解答 Paper Formatter 的常见问题。

## 🚀 安装与启动

### Q1: 系统要求是什么？
**A:** Paper Formatter 支持以下系统：
- **Windows**: Windows 10/11 (64位)
- **macOS**: macOS 10.15+ (Catalina 及以上)
- **Linux**: Ubuntu 18.04+, CentOS 7+, 或其他主流发行版
- **内存**: 最低 4GB，推荐 8GB
- **存储**: 最低 500MB 可用空间

### Q2: 安装失败怎么办？
**A:** 尝试以下步骤：
1. **检查系统要求**：确保系统满足最低要求
2. **以管理员身份运行**：Windows 用户右键点击安装程序选择"以管理员身份运行"
3. **关闭杀毒软件**：临时关闭杀毒软件再安装
4. **清理临时文件**：删除旧的安装文件重新下载
5. **查看日志文件**：安装日志位于 `%TEMP%\PaperFormatterInstall.log`

### Q3: 启动时崩溃或无响应？
**A:** 尝试以下解决方案：
1. **重启应用**：关闭后重新启动
2. **更新显卡驱动**：更新到最新版本
3. **检查依赖**：确保安装了 .NET Framework 或 VC++ Redistributable
4. **安全模式启动**：使用 `--safe-mode` 参数启动
5. **查看错误日志**：日志位于 `%APPDATA%\Paper Formatter\logs\` (Windows) 或 `~/.paper-formatter/logs/` (Linux/macOS)

## 📄 文档处理

### Q4: 支持哪些文档格式？
**A:** 目前支持：
- **Microsoft Word**: .docx, .doc
- **纯文本**: .txt
- **Markdown**: .md
- **其他格式**：通过插件支持（如 .pdf, .rtf）

### Q5: 文档打开失败怎么办？
**A:** 检查以下问题：
1. **文件权限**：确保有读取权限
2. **文件损坏**：尝试用其他程序打开
3. **文件锁定**：关闭其他程序中的该文件
4. **路径问题**：避免使用特殊字符或过长的路径
5. **编码问题**：确保文件使用 UTF-8 编码

### Q6: 格式化后样式丢失？
**A:** 可能的原因和解决方案：
1. **模板不匹配**：选择正确的模板
2. **字体缺失**：安装缺失的字体
3. **样式冲突**：检查文档中的自定义样式
4. **版本兼容性**：确保使用最新版本
5. **导出设置**：调整导出选项

## 🎨 格式与模板

### Q7: 如何创建自定义模板？
**A:** 创建自定义模板的步骤：
1. 点击"设置" → "模板管理"
2. 点击"新建模板"
3. 配置页面、字体、段落等设置
4. 保存模板并命名
5. 在格式化时选择自定义模板

### Q8: 模板不适用于我的文档？
**A:** 尝试以下方法：
1. **检查文档结构**：确保文档有正确的标题结构
2. **调整模板参数**：微调模板设置
3. **使用基础模板**：从简单模板开始
4. **分段格式化**：分章节格式化文档
5. **联系支持**：提供文档示例以便调试

### Q9: 如何批量应用模板？
**A:** 批量处理步骤：
1. 点击"批量处理"按钮或使用 `Ctrl+B`
2. 选择包含文档的文件夹
3. 选择输出模板
4. 配置输出选项
5. 点击"开始处理"

## 🔤 字体管理

### Q10: 字体下载失败？
**A:** 解决方案：
1. **检查网络连接**：确保网络正常
2. **更换下载源**：在设置中更换字体源
3. **手动安装字体**：从其他来源下载并安装
4. **使用系统字体**：使用已安装的系统字体
5. **清理字体缓存**：删除字体缓存重新下载

### Q11: 中文字体显示异常？
**A:** 解决方法：
1. **安装中文字体**：确保系统中安装了中文字体
2. **设置默认字体**：在设置中指定中文字体
3. **检查编码**：确保文档使用正确的编码
4. **更新字体库**：更新到最新字体库
5. **使用兼容字体**：使用更兼容的字体如"微软雅黑"

## 💾 备份与恢复

### Q12: 备份文件在哪里？
**A:** 备份文件默认位置：
- **Windows**: `%APPDATA%\Paper Formatter\backups\`
- **macOS**: `~/Library/Application Support/Paper Formatter/backups/`
- **Linux**: `~/.paper-formatter/backups/`

可以在设置中更改备份位置。

### Q13: 如何恢复备份？
**A:** 恢复步骤：
1. 点击"文件" → "恢复备份"
2. 选择备份文件或时间点
3. 预览备份内容
4. 选择恢复选项（完全恢复或部分恢复）
5. 确认恢复

### Q14: 备份占用太多空间？
**A:** 管理备份空间：
1. **调整备份策略**：减少备份频率
2. **设置保留数量**：限制保留的备份数量
3. **清理旧备份**：手动删除不需要的备份
4. **启用压缩**：启用备份文件压缩
5. **更改存储位置**：移动到更大容量的驱动器

## ⚡ 性能问题

### Q15: 处理大型文档很慢？
**A:** 优化建议：
1. **关闭其他程序**：释放系统资源
2. **增加内存分配**：在设置中增加内存限制
3. **分段处理**：分章节处理大型文档
4. **使用 SSD**：将文档放在 SSD 上
5. **更新硬件**：考虑升级内存或 CPU

### Q16: 内存占用过高？
**A:** 减少内存使用：
1. **关闭预览**：处理时关闭实时预览
2. **减少历史记录**：限制保存的历史版本数量
3. **清理缓存**：定期清理临时文件
4. **优化设置**：调整性能相关设置
5. **重启应用**：定期重启释放内存

## 🔌 插件与扩展

### Q17: 如何安装插件？
**A:** 插件安装方法：
1. 下载插件文件（.ppx 格式）
2. 点击"设置" → "插件管理"
3. 点击"安装插件"
4. 选择插件文件
5. 重启应用生效

### Q18: 插件不兼容怎么办？
**A:** 解决方法：
1. **检查版本**：确保插件与软件版本兼容
2. **更新插件**：下载最新版本插件
3. **联系开发者**：报告兼容性问题
4. **使用替代插件**：寻找功能相似的插件
5. **禁用插件**：暂时禁用有问题的插件

## 🔧 故障排除

### Q19: 程序崩溃如何收集信息？
**A:** 收集调试信息：
1. **查看日志文件**：位置见 Q3
2. **启用调试模式**：启动时添加 `--debug` 参数
3. **截图错误信息**：保存错误对话框截图
4. **记录操作步骤**：记录崩溃前的操作
5. **提交问题报告**：包含以上信息提交到 [GitHub Issues](https://github.com/taobaoaz/paper-formatter/issues)

### Q20: 如何重置设置？
**A:** 重置方法：
1. **通过界面重置**：设置 → 高级 → 重置所有设置
2. **手动删除配置**：删除配置文件：
   - Windows: `%APPDATA%\Paper Formatter\config.json`
   - macOS: `~/Library/Application Support/Paper Formatter/config.json`
   - Linux: `~/.paper-formatter/config.json`
3. **重新安装**：卸载后重新安装

## 📱 跨平台问题

### Q21: Windows 和 macOS 版本功能有差异吗？
**A:** 核心功能相同，但有以下差异：
- **界面风格**：遵循各自平台的设计规范
- **快捷键**：使用平台标准的快捷键
- **文件路径**：处理平台特定的路径格式
- **系统集成**：与各自系统的集成方式不同

### Q22: Linux 版本的特殊问题？
**A:** Linux 用户需注意：
- **字体管理**：可能需要手动安装字体
- **权限问题**：确保有足够的文件权限
- **依赖库**：可能需要安装额外的库
- **桌面环境**：不同桌面环境可能有不同表现
